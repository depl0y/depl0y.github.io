Filter photos in Photos using AppleScript

Wim Haanstra
This post was created on April 29, 2015.
applescript photos filter osx

I have been cleaning up my Photos library lately. I have lots of screenshots in my library somehow (my wife and I sync to the same library) and I wanted to clean those up a bit. So at first I wanted to create a Smart Album and just let it filter on the dimensions of the photos.

Well, that didn’t work. You somehow can’t filter on height and width of photos in Photos. Now I have been playing with AppleScript every couple of months and this should be easily doable using AppleScript.

So here is a script that filters your Photos in an album and then puts them in another album once they match dimensions.

tell application "Photos"
	
	-- The name of the album you want to search for screenshots
	set source_album_name to "My Source Album"

	-- The name of the album where you want to add the screenshots to
	set target_album_name to "Screenshots"
	
	-- resolutions: iPhone 4, iPhone 5S, iPhone 6 defined as { width x height }
	set target_resolutions to {{480, 640}, {640, 1136}, {750, 1334}}
	
	-- If the 'target_album_name' does not exist as an Album, it will be created
	if not (exists container named target_album_name) then
		make new album named target_album_name
	end if
	
	-- Fetch the album you want to search
	set target_album to container target_album_name

	-- Fetch the album you want to store the images in
	set source_album to container source_album_name
	
	repeat with resolution in target_resolutions
		set target_width to item 1 of resolution
		set target_height to item 2 of resolution
		
		add (get every media item in source_album whose width is equal to target_width and height is equal to target_height) to target_album
	end repeat
	
end tell

There is one problem though.

It’s still not flawless, because sometimes I get some un-googlable error.

error "Photos got an error: AppleEvent handler failed." number -10000

This error happens from time to time (it is consistent, this occurs on the same albums every time). First I thought it was because of the amount of pictures I have in my library, but sometimes it occurs on an album with 1.000 photos and then it works flawlessly on an album with 3.500 pictures.
