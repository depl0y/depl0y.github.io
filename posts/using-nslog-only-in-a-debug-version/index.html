Using NSLog only in a DEBUG version

Wim Haanstra
This post was created on October 15, 2010.
objC

I have been using this in a couple of projects. It is a method to only perform NSLog logging when you are using a DEBUG build. There are more than enough reasons why you don't want to use NSLog in your release builds.

First of all, its all about performance. I noticed that when you use NSLog a lot it decreases performance significantly. Ofcourse you can solve this in more ways, but I found this the easiest one.

Setting the Preprocessor Macro

First you need to set a Preprocessor macro for your debug build. You can do this by double clicking on your target application. The Target Info window will popup and you need to make sure that you choose the Debug configuration.

The just perform a search in that window for Preprocessor Macros. Once you found that, make sure you add the value DEBUG=1.

Adding some code

Now we are going to add some code to your applications prefix file. Just look through your applications' sources for a file called something like YourApplication_Prefix.pch. Once you find that, open up that file and use the following code:

1 #ifdef DEBUG
2     #define NSLog(format, ...) NSLog(format, ## __VA_ARGS__)
3 #else
4     #define NSLog(format, ...)
5 #endif

This code overrides the normal NSLog functionality. And as you can see it will only execute a real NSLog when DEBUG is not set (so, your release build).
