HomeWizard API calls

Wim Haanstra
This post was created on April 14, 2014.
homewizard API

I am currently working on getting my HomeWizard up and running with a nice iPad application I am building. I only read settings from the HomeWizard, so I am focusing on reading the sensors and switching lights, turning up heat, etc. If you want to add sensors or change settings, just sniff the API calls using a network sniffer. If there is any interest, I might update this article to include them.

http://<homewizard-ip>/<password>/get-sensors
Get all the sensors, switches, smoke detectors, etc connected to your HomeWizard.

http://<homewizard-ip>/<password>/get-status
The same response as get-sensors, but in a shorter format. I use this for updating the status of buttons, dimmers, etc.

http://<homewizard-ip>/<password>/sw/<switch id>/[on/off]
Turn a switch with id on or off.

http://<homewizard-ip>/<password>/sw/dim/<switch id>/[0..100]
Dim a dimmer to a [0..100] value. According to the documentation this should be [0..255], but this is giving me weird results.

http://<homewizard-ip>/<password>/sw/<switch id>/[on/off]/[0..360]/[0..100]/[0..100]
Control a Hue light. Where the on or off switches the light on or off. The other values are: Hue, Saturation and Brightness in that order.

http://<homewizard-ip>/<password>/gp/get/<scene id>
Get the details of a scene, like the buttons and their statuses.

http://<homewizard-ip>/<password>/gp/get/<scene id>
Get the details of a scene, like the buttons and their statuses.

http://<homewizard-ip>/<password>/gp/<scene id>/[on/off]
Turn a scene with id on or off.

http://<homewizard-ip>/<password>/sf/<somfy id>/[down/up/stop]
Control a Somfy device. Up goes up, Down goes down and Stop stops the current actions OR sends the Somfy to the predefined state.

http://<homewizard-ip>/<password>/el/graph/<EnergyLink ID>/[day/week/month/year]
Get values to generate graphs for the specified EnergyLink.

http://<homewizard-ip>/<password>/te/graph/<Thermometer ID>/[day/week/month/year]
Get values to generate graphs for the specified thermometer. So this what I currently have, something I am building at the moment is a control panel for every switch in the house, including some statistics about power usage, outside temperatures, etc.
