Getting Windows 8, Visual Studio and IIS to play nice

Wim Haanstra
This post was created on October 21, 2013.
Windows 8 Visual Studio IIS

I have upgrading to Windows 8 a couple of times now, but I had one problem with it. We use IIS on our laptops to host the project we are currently working on and use Visual Studio to debug right on it. Since Windows 8 does not give users FULL administrator rights anymore, you always had to start Visual Studio as an administrator. This screwed up double clicking your solution files and have enough permissions to connect to IIS for debugging.

Finally I have found a solution for this problem and I can finally start using Windows 8(.1). So you want to use Visual Studio as an administrator and connect correctly to IIS every time you start it? Follow these steps:

Find your devenv.exe

For x64 systems this can be found here (Visual Studio 2013 on x64 Windows 8.1):

1 C:\Program Files (x86)\Microsoft Visual Studio 12.0\Common7\IDE\devenv.exe

Do some compatibility troubleshooting

Right click devenv.exe and click “Troubleshoot compatibility”.

It will start looking for a solution online. Ignore that.

Choose “Troubleshoot program”.

It shows you a couple of options to fix this issue. Select the one saying “The program requires additional permissions”.

Test it out

In this wizard, click ‘next’ and it will ask you to test your fix. Click ‘Test the program’ and it will start Visual Studio.

Close it for now and click “next” in the wizard. We are assuming this fixes our problem and we click “Yes, save these settings for this program”.

Finished

Yay, we are done now. Now Visual Studio ALWAYS starts in Administrator mode and can connect to IIS without problems. It makes my life so much easier.
